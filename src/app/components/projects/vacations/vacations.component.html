This project is the third project as part of the Full Stack course I took. The first two were a html5 and jQuery projects which I am not considering as significant as this one and as the eCommerce project because they did not include server-side architecture or HTTP rest API calls. Those two projects have been plain learning of the languages. Although that is important, they were not close to what is around the web development community.


Even though I did not receive the high score that I wanted, I present this project as part of my portfolio for two reasons:

  • I did not do it alone - I got help from my coursemates. They helped me complete it and understand the thoughts behind every action.
  • it was probably one of the first times I worked hard to create something of my own.


    • This project was about creating a website that will give his users the option to follow changes in vacations they might be interested in. Our goal here was to use the website database to store the "vacations" and the data of the users so they would have the ability to follow vacations and enquire about them when the time is right. In addition, we also managed an admin side of the website that will allow the admin to add, change details or delete vacations based on criteria.


    Why didn't I receive a high score on this project?

  • UX/UI design - was not good enough to attract people.
  • Insufficient use of methods - the following vacation mechanism in this project was accomplished with the JavaScript sort array method. At the time of writing the project, I had a problem learning this method, and actions in the project were affected, beginning with the database and ending with UX/UI design.
  • A required aspect of the project was using socket I.O. communication which will allow sending changes of data to the user without making him refresh the page to receive the changes that happened. I had failed to make this aspect.
